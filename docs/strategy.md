## Thesis

The objective is not to build a services business.
The objective is to build a **proprietary intelligence layer on technology-driven value creation**, using services as the data acquisition engine.

> Positioning:
> "We predict and deliver technology-driven value creation outcomes in mid-market and PE-backed companies."

> Differentiator:
> "Technology DD with **quantified financial impact**." Most DD reports say "the architecture has technical debt." Ours says "GBP 2.4M of technical debt, GBP 800K to remediate over 18 months, 3.2x ROI."

## 1. Entry Point: Tech Due Diligence (DD)

### Why DD
- Highest **data density per engagement**
- Fast sales cycle (2–4 weeks)
- Executive buyer (PE operating partner, deal lead, CFO)
- Natural gateway into post-deal work

### Offer (v1)

> **"Technology DD with quantified risk and value creation plan in 10–15 days"**

### Outputs
- Architecture + team assessment
- Cost-to-fix estimate (quantified in GBP, phased)
- Risk scoring (standardized 1–5 across 7 dimensions)
- Value creation roadmap (90-day view)
- IC-ready executive summary

### Hidden Objective
Capture **structured, comparable data** across all engagements.

## 2. Expansion Path (Service → Platform)

- Pricing power increases significantly

- Automated risk + cost reports
- Portfolio dashboards

## 3. Proprietary Data Model

### Objective
Enable the system to answer:
> "For a company like this, what will happen if we do X?"

### Core Schema

**1. Company:** sector, revenue band, geography, stage (PE, Series B, etc.)

**2. Tech Stack:** cloud providers, architecture type, core systems, tooling (CI/CD, observability)

**3. Team / Org:** headcount, composition ratios, seniority, outsourcing %

**4. Problem Taxonomy:** Standardized categories — cost inefficiency, scalability bottleneck, data fragmentation, security exposure, delivery dysfunction. Each tagged with severity (1–5) and affected systems.

**5. Intervention:** type, scope, duration, cost

**6. Outcome (Critical Layer):** cost savings (GBP, %), revenue uplift (GBP, %), performance delta, time to impact (days)

**7. Prediction vs Reality (Moat Layer):** predicted cost/timeline/outcome → actual results → variance

### Derived Intelligence
- **Benchmarks:** cloud cost/revenue ratios, team efficiency, cost-to-fix distributions
- **Patterns:** stack → failure probability, org structure → delivery outcomes
- **ROI Maps:** intervention → expected return distribution
- **Risk Scoring:** composite (architecture + team + execution risk)

### Network Effects Flywheel
More DD engagements → More structured data → Better benchmarks + predictions → Better outcomes → Stronger credibility → More engagements

> Each client improves outcomes for all future clients.

## 5. How PE Firms Actually Hire DD Providers

### Discovery channels (in order of importance)
1. **Operating partner networks** — "Little black book." Dominant channel. Relationship sale
2. **Curated platforms** — BluWave (invitation-only, vetted, 24hr matching)
3. **Expert networks** — GLG, AlphaSights, Third Bridge, Guidepoint (gateway, not primary)
4. **Consulting firm relationships** — Bain, EY-Parthenon (large-cap only)
5. **Referrals between PE firms** — Strong report on one deal → recommended for next

### What is NOT a significant channel
- Google search
- Cold outbound / LinkedIn prospecting (low conversion)
- Advertising of any kind
- SEO / domain keywords

### Implication
Every minute spent on SEO, domain optimization, or marketing is a minute not spent on relationship building. The brand/website exists for credibility validation after a referral, not for lead generation.

## 10. Operating Principles

1. **Standardization > Completeness** — 20 consistent variables > 100 inconsistent ones
2. **Quantification > Narrative** — No numbers → no dataset → no IP
3. **Comparability > Depth** — Cross-client comparability is the asset
4. **Prediction Tracking is Mandatory** — Every engagement: explicit predictions + measured outcomes
5. **The IC memo is your client** — Make the DD report so clean it copy-pastes into the memo

## 13. Trust-Building Sequence (ranked by impact)

The website is necessary but not sufficient. Trust in PE DD is built through these vectors, in order of impact:

### Tier 1 — Proof of work (highest impact)
1. **First real DD engagement** — even one, even discounted. A completed engagement with a testimonial from a PE professional outweighs everything else. Price first 1–2 at cost (GBP 10–15K) in exchange for reference + case study
2. **Downloadable sample DD report** — full mock DD as a professional PDF (20+ pages, anonymised, clearly labelled "illustrative"). An operating partner downloads it, flips through, and judges quality. Not mockups on a website — an actual document
3. **Named endorsement from a PE professional** — one quote from Samir, Victor, or Lorenzo after conversations would outweigh all current testimonials combined. Current testimonials are from operators/founders, not PE buyers

### Tier 2 — Published thinking (builds credibility at scale)
4. **Public analysis using the framework** — Substack post where Pedro runs the DD framework against a real public company. Proof of thinking, not proof of formatting
5. **LinkedIn content cadence** — 2–3 posts/week demonstrating PE-relevant pattern recognition. "What I'd check" and "cost quantification" posts
6. **First long-form article** — "Common Technology Risk Patterns in PE-Backed Mid-Market Companies" based on Velkan portfolio data

### Tier 3 — Institutional signals (slower but durable)
7. **UK Private Capital (BVCA) membership** — Professional Services category. Signals legitimacy, opens 50+ events/year
8. **BluWave registration** — invitation-only PE provider network. Highest-leverage platform
9. **Advisory network profiles** — GLG, AlphaSights, Guidepoint, Third Bridge. Bridge income + PE exposure
10. **UK Ltd formation** — institutional contracting vehicle

### Tier 4 — Website and materials (validation, not generation)
11. **Website with sample deliverables** — shows methodology depth. Credibility validation after referral
12. **One-pager** — physical handout at events and meetings
13. **CV and LinkedIn** — consistent positioning across all surfaces

### Key insight
The website is at diminishing returns. The next unit of trust comes from **conversations, content, and the first engagement** — not more CSS. The mock DD report (downloadable PDF) and the first LinkedIn post move the needle more than further website refinement.
